# MVP

Minimal Viable Product:
- The open tech spec is written as a to-do list of tasks with checkboxes.
- The lines of the open tech spec are sorted in order priority.
- Always read the open tech spec starting from line 1.
- Always focus on the first unchecked action item.
- Always implement a minimal test that fails the action item.
- The test failure message isolates the root cause with 1 to 256 characters.
- Always apply the 80/20 principle of only solving most of the problem in a simple phase.
- Always factor architecture to reduce context.
- Always solve a problem in phases.
- Always audit each phase for a mistake.
- Always reduce the problem to conventional algorithms and tools.
- Always write self-documenting code without comments.
- Always implement a simple, professional Next.js 16 solution.
- Always limit files sizes between 1 and 256 lines.
- Always iterate until all `/git_hooks/pre-push` passes.
- After passing, always check the box at that unchecked action item.
- Always limit a chat response between 1 to 256 characters.
